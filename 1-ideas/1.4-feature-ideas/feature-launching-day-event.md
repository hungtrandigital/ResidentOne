# Feature Idea: Launching Day Event for Residents & Sellers

**Date:** 2025-12-13  
**Last Updated:** 2025-12-13  
**Status:** Feature Idea Analysis  
**Type:** Feature Addition (Onboarding Process Enhancement)

---

## Executive Summary

**Idea:** Add a Launching Day event to the onboarding process where residents and sellers (tiểu thương) are invited to learn about the marketplace and community platform.

**Recommendation:** ✅ **STRONGLY RECOMMENDED** - High impact on user adoption and critical mass achievement

**Key Benefits:**
- Accelerates user adoption and critical mass
- Builds community engagement from day one
- Demonstrates value proposition directly
- Creates word-of-mouth momentum
- Addresses key risk: user adoption

**Key Considerations:**
- Additional cost: VND 2-5M per building
- Requires coordination with BQL/BQT
- Timing: Should be part of Phase 6 (Launch) in onboarding process

---

## 1. Idea Description

### 1.1 Concept

**Launching Day Event:**
- Physical or hybrid event (physical preferred for community building)
- Target audience: Residents + Sellers (tiểu thương)
- Purpose: Introduce platform, demonstrate value, onboard users
- Timing: During Phase 6 (Launch) of onboarding process
- Duration: 2-4 hours (afternoon/evening event)

**Event Components:**
1. **Platform Introduction** (30-45 min)
   - Platform overview and value proposition
   - Key features demonstration
   - Trust & safety explanation
   - Resident-owned concept

2. **Live Demo** (30-45 min)
   - Live platform walkthrough
   - Registration process
   - How to list products
   - How to search and buy
   - Q&A session

3. **Onboarding Support** (60-90 min)
   - Help residents register on-site
   - Help sellers create first listings
   - Answer questions
   - Provide tutorials

4. **Community Building** (30-45 min)
   - Networking opportunity
   - Community ambassador introduction
   - Future events planning
   - Feedback collection

   - Provide tutorials

- No change to overall timeline (4-6 weeks)

- May actually accelerate post-launch adoption

---

### 4.2 Event Format Options

**Option 1: Physical Event (Recommended)**
- **Venue:** Building common area (lobby, community room)
- **Format:** In-person event with live demo
- **Pros:** 
  - Strongest community building
  - Best for trust building
  - Highest engagement
- **Cons:**
  - Higher cost (venue, refreshments)
  - Requires coordination with BQL/BQT
  - Weather/logistics dependent

**Option 2: Hybrid Event**
- **Venue:** Building common area + online streaming
- **Format:** In-person + online participation
- **Pros:**
  - Reaches more residents
  - Lower cost than full physical
  - Flexible participation
- **Cons:**
  - Less community building than full physical
  - Technical setup required

**Option 3: Online Event**
- **Venue:** Online (Zoom, Teams, etc.)
- **Format:** Virtual event with screen sharing
- **Pros:**
  - Lowest cost
  - Reaches all residents
  - No logistics issues
- **Cons:**
  - Weakest community building
  - Lower engagement
  - Less trust building

**Recommendation:** **Option 1 (Physical Event)** for first 10-20 buildings, then consider hybrid for scale

---

## 6. Risks & Mitigation

### 6.1 Risks

**Risk 1: Low Attendance**
- **Impact:** High - Event fails to achieve adoption goals
- **Probability:** Medium
- **Mitigation:**
  - Strong promotion (BQL/BQT support, flyers, announcements)
  - Incentives (early adopter benefits, refreshments)
  - Multiple communication channels
  - Community ambassador support

**Risk 2: Technical Issues During Event**
- **Impact:** Medium - Damages credibility
- **Probability:** Low
- **Mitigation:**
  - Thorough testing before event
  - Backup plans (offline demo, screenshots)
  - Technical support on-site

**Risk 3: Cost Overrun**
- **Impact:** Medium - Affects ROI
- **Probability:** Medium
- **Mitigation:**
  - Use building common area (low cost)
  - Standardize event format
  - Track costs per building
  - Optimize based on learnings

**Risk 4: BQL/BQT Resistance**
- **Impact:** High - Event cannot proceed
- **Probability:** Low
- **Mitigation:**
  - Emphasize value to residents (builds BQL/BQT credibility)
  - Offer to co-host with BQL/BQT
  - Provide event support to BQL/BQT
