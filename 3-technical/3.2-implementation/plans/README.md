# Implementation Plans

This directory contains all implementation plans organized by lifecycle stage.

## Structure

- **[active/](active/)** - Currently active implementation plans
- **[completed/](completed/)** - Completed plans (organized by month: `YYYY-MM/`)
- **[archived/](archived/)** - Archived plans (organized by date: `YYYY-MM-DD/`)
- **[epics/](epics/)** - Epic-level plans and documentation

## Plan Lifecycle

1. **Active** → Plans being worked on
2. **Completed** → Implementation done, moved to `completed/[YYYY-MM]/`
3. **Archived** → After 3 months, moved to `archived/[YYYY-MM-DD]/` by @docs-guardian

## Active Plans

## Plan Metadata

All plans must include metadata:
- `status`: active | completed | archived
- `type`: feature | fix | refactor | epic
- `priority`: high | medium | low
- `created`: YYYY-MM-DD
- `updated`: YYYY-MM-DD
- `epic`: epic name or `-`
- `system`: system name or `cross-system`
